# Lec1

!!! abstract

    - AVL 树：自调整的二叉搜索树（BST），任意节点的左右子树高度差不超过 1。
    - Splay（伸展）树：最近一次搜索或新增的内容会被移至根。
    - Amortized 摊还分析：求操作序列中所有操作的平均时间，评价操作的代价。用于特定操作非常慢，但一般操作很快的算法。

## 摊还分析

摊还分析用于特定操作非常慢，但一般操作很快的算法，比如：哈希表、并查集、伸展树。

摊还分析不使用概率知识。

??? example "动态数组的扩张"

    动态数组插入的最坏情况是 $O(n)$（创建新数组并拷贝 $n$ 个元素），简单分析得出插入操作上界为 $n*O(n)=O(n^2)$。然而并不是每次插入都是 $O(n)$，这个上界并不好。

    采用摊还分析，计算如下：

    $$
    \frac{(1+1+1+\dots)+(1+2+4+\dots)}{n}=\frac{n+\lfloor\log_2(n-1)\rfloor+1}{n}\leq3
    $$

    得到插入的平均时间复杂度为 $O(1)$。

??? example "红黑树的插入"

    ??? note "回顾红黑树"

        红黑树也是一种平衡搜索树，保证最坏情况动态集合操作时间复杂度 $O(\log_2 n)$。红黑树保证**没有一条路径会比其他路径长出 2 倍**。

        红黑树每个节点增加一个存储位表示节点的颜色。根节点是黑的，叶节点是黑的。如果一个节点是红的，则两个子节点都是黑的。每个节点到其所有后代叶节点的路径上，都包含相同数目的黑色节点，称为黑高 $bh(x)$。

        具有 $n$ 个内部节点的红黑树高度最多为 $2\lg(n+1)$。
