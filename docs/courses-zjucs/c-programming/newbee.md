## 学习 C 语言

我自己是靠把《C Primer Plus》啃完完成 C 语言入门的，因此我也倾向于向身边的人推荐通过这本书入门。可能 xpy 们会畏惧这么一大本书，但是，C 语言真的会是你很重要的语言基础。如果你将要学习计算机相关专业，在这上面投入时间是值得的。而且，这本书写得十分通俗易懂，阅读难度其实很低，只需要你努力去看即可。如果你能啃下这本书，面对后面计算机专业课的那些教材你也不会有丝毫畏惧了。

当然也有更薄的选择，我推荐《Linux C 一站式学习》，这本书对 C 语言基本原理的讲解十分精炼而透彻。摘一下这本书前言中的一段话，或许也是一些同学们的痛苦所在：

> 比如我刚上大学的时候，大一上半学期就被逼着学C语言，其实C语言是一门很难的编程语言，不懂编译原理、操作系统和计算机体系结构根本不可能学明白，那半个学期自然就浪费掉了。当时几乎所有学校的计算机相关专业都是这样，大一上来就学C语言，有的学校更疯狂，上来就学C++，导致大多数学生都以为自己会C语言，但其实都是半吊子水平，到真正写代码的时候经常为一个Bug搞得焦头烂额，却没有机会再系统地学一遍C语言，因为在学校看来，C语言课早在大一就给你“上完了”，就像一顿饭已经吃完了，不管你吃饱没吃饱，不会再让你重吃一遍了。显而易见，如果要认真地对这些课程做优化，的确是有很多水份可以挤的。

## 去哪里查找资料

或许劝大多数同学扔掉百度和 CSDN 转而用 Google 和 StackOverflow 会是很难的事情，第一道坎就是新生可能不太会翻墙。在有的同学看来，国外网站对自己的帮助不如 CSDN 大，这取决于个人的学习习惯。相对来说，国外社区的资料和讨论更加专业和深入，国内社区的氛围非常之差，鱼龙混杂。举例：

- 在百度上搜编程问题，大概率排名前面的是百度教程这种纯粹的垃圾。
- 大量网站机翻国外的资料。
- CSDN 各种的关注查看、登陆查看、收费等，其中很多文章质量极低或是严重过时，存在误导可能。

如果可能的话，请尝试更多地转向国外网站寻找资料。暂时的转变的痛苦会为你带来长久的收益。
