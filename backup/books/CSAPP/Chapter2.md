# Ch2. 信息的表示和处理

## 2.1 信息存储

!!! abstract

    本章主要讲解一些和字节编码有关的基础知识，为下一章阅读汇编代码和二进制数据奠定基础。

- 十六进制表示法：
    - 熟记 `A, C, F` 的二进制表示即可。
    - 练习：对十六进制数或其他进制数做加减运算。如何借位？
- 字长
    - 我们常说的“32 位”等，指的是计算机的字长，即指针变量的标称长度。
    - 我们说程序是“32 位”或“64 位”时，指的是程序编译时的配置。
    - gcc 编译时使用 `-m32` 和 `-m64` 选项可以指定字长。
- 数据大小
    - C 标准为不同数据类型范围设置了下界，却没有设置上界。
    - 使用 C99 引入的 `int32_t` 和 `int64_t` 等类型可以保证数据大小。
    - 大部分数据类型是有符号的，`char` 类型未作规定，大部分情况下程序行为不会因此不同。
- 寻址和字节顺序
    - 大多数机器上，对象的地址是所占字节的最小的地址。
    - Intel 机器一般采用小端法：数字的最低位字节放在最前面。比如 `0x123456` 在内存中顺序放置为 `56 34 12`。
    - Android 和 iOS 都只能采用小端法。它们通常运行在 ARM 微处理器上，其硬件可以配置为大端/小端两种模式。
    - 在这些方面，字节顺序会产生影响：通过网络传送二进制数据、检查机器级程序和编写规避**类型系统**的程序时。

练习：书本提供了 `show_bytes` 例程，可以用于检查机器的字节顺序。

!!! note

 指针的强制类型转换并不会改变指针的值，只是告诉**编译器**以另一种类型看待该指针。根据书本的描述，指针和类型系统都由编译器所控制。

- 布尔代数
    - 掌握简单的四种布尔运算即可。深入的学习在“离散数学及其应用”课程中。
    - 位向量常用于为集合编码。
    - 练习题提到了利用 `a^a=0` 性质编写的三变量交换程序，这是一个技巧，但请不要用在生产环境中。它可能会遇到问题，比如对指向同一位置的指针进行交换时。
    - 需要掌握的技能是：如何根据已有的布尔函数，构造所需的布尔函数。可以通过观察真值表寻找关系，更重要的是学会使用两种方法解析其中的布尔表达式。
- 移位运算
    - 简记：右移时，逻辑补 0，算数补最高位。移动时取模。
    - 移位运算的优先级低于加减运算，高于逻辑运算。
